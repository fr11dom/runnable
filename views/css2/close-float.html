再看闭合浮动

浮动对于布局来讲是个好东西，但是副作用也不少。比如浮动后父元素高度变0，低版本IE下bug等等。

记得刚开始用的最多的就是空白div办法，简单易懂可是一直觉得放个空div在这里简直太奇葩。

 .clear{clear: both}
 <div>
    <div class="left">...</div>
    <div class="right">...</div>
    <div class="clear"></div>
 </div>

然后网上找到另一种觉得不错的方法。拿来主义,没好好理解其实现原理。

    .clearfix:after {
	  content:"."; 
	  display:block; 
	  height:0; 
	  visibility:hidden; 
	  clear:both; 
    }
    .clearfix { *zoom:1; }
    <div class="clearfix">
        <div class="left">...</div>
        <div class="right">...</div>
    </div>

其实吧，也好理解，新的方法还是依靠clear:both来闭合浮动，其他代码都是为了隐藏生成的content而已，而*zoom则是直接触发IE的hasLayout来闭合浮动
