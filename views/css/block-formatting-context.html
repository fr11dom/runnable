Block Formatting Context

CSS背后居然站着BFC这种大神！

关于BFC很多IT博主都崇拜过，作为一个菜鸟今天终于鼓起了勇气，我也来走进大神、模仿大神看是不是能提升我CSS的功力！

google后首先发现的是这篇文章Block formatting contexts，我觉得翻译的真用心，所以也用心看了下。

我得到这么些信息：

  1. BFC是属于正常排版（Normal flow）,因此它会一个接一个的垂直摆放。
  2. 在同一个BFC里，相邻块margin的top和bottom会折叠。
  3. 在BFC里，每个块会紧贴父块的左侧，浮动也不会改变,除非创建新的BFC。
  4. 创建新的BFC是要有条件的。

创建新的BFC的具体条件：

  1. 能产生浮动框属性，float: left | right | inherit
  2. 产生绝对定位框的属性，position: absolute | fixed
  3. 产生非块框的块容器的属性，display: inline-blocks | table-cells | table-captions
  4. overflow属性的非'visible'块框，overflow: hidden | scroll | auto | inherit

嗯，这货的作用以后慢慢发掘，此篇原理放此备查！
